1. 객체지향의 기본
클래스와 객체 (💡): 클래스는 데이터를 담는 '설계도'이고, 객체(인스턴스)는 그 설계도를 바탕으로 메모리에 할당된 '실체'입니다.

유니티 적용: 몬스터의 능력치와 행동 패턴을 담은 설계도(클래스)를 만들고, 실제 게임 씬에 여러 마리의 몬스터를 소환(Instantiate)하는 기본 원리가 됩니다.

생성자와 종료자: 객체가 생성될 때(생성자)와 소멸될 때(종료자) 자동으로 호출되는 메소드입니다. 주로 객체의 초기 상태를 설정할 때 사용합니다.

2. 메모리와 참조
정적(static) 필드와 메소드 (💡): 개별 인스턴스가 아닌 클래스 자체에 소속되어 메모리에 단 하나만 존재하게 됩니다.

유니티 적용: 전체 게임 점수 관리, 게임 매니저(GameManager) 등 게임 내에 유일하게 존재하며 어디서든 접근해야 하는 데이터를 다룰 때 필수적입니다.

얕은 복사와 깊은 복사: 객체를 복사할 때 주소(참조)만 복사하여 원본을 공유할지(얕은 복사), 아니면 메모리에 새로운 공간을 할당해 실제 데이터를 똑같이 베껴 쓸지(깊은 복사)의 차이입니다.

this 및 this() 키워드: this는 객체 자기 자신을 가리키며, this()는 자기 자신의 다른 생성자를 호출할 때 사용합니다.

3. 객체지향의 꽃, 상속과 다형성
접근 제한자 (💡): public, private, protected 등을 사용하여 외부에서 데이터에 접근하는 것을 허용하거나 차단(캡슐화)합니다.

유니티 적용: public으로 선언한 변수는 유니티 에디터의 인스펙터 창에 노출되어 엔진에서 직접 수치를 조절할 수 있습니다.

상속과 base (💡): 기존 클래스(기반)의 기능과 속성을 물려받아 새로운 클래스(파생)를 만드는 기능입니다. base는 부모 클래스를 가리킵니다.

유니티 적용: 유니티에서 여러분이 작성하는 거의 모든 스크립트는 엔진에 이미 구현된 MonoBehaviour라는 부모 클래스를 상속받아 만들어집니다.

형변환 (is, as) (💡): 부모와 자식 클래스 간의 형태를 변환하고 확인합니다.

유니티 적용: 플레이어의 칼에 부딪힌 객체가 '적군'인지, '벽'인지, '아이템'인지 판별할 때 정말 자주 사용합니다.

오버라이딩과 다형성 (💡): 부모 클래스의 메소드(virtual)를 자식 클래스에서 입맛에 맞게 재정의(override)하는 기능입니다.

유니티 적용: 같은 '공격' 명령을 내려도 전사는 칼을 휘두르고 마법사는 파이어볼을 쏘도록 다르게 구현할 때 쓰입니다.

4. 클래스의 확장 및 데이터 구조
메소드 숨기기 / 클래스 봉인 (sealed): 상속을 더 이상 못하게 막거나, 부모의 메소드를 강제로 가리는 고급 기법입니다.

중첩 및 분할 클래스 (partial): 클래스 안에 또 다른 클래스를 만들거나, 내용이 너무 긴 하나의 클래스를 여러 파일로 나누어 작성하는 방법입니다.

확장 메소드: 기존 클래스의 원본 코드를 건드리지 않고도 새로운 기능을 덧붙일 수 있는 문법입니다.

구조체 (struct) (💡): 클래스와 비슷하게 생겼지만, 참조 형식(힙 메모리)이 아닌 값 형식(스택 메모리)으로 가볍게 동작합니다.

유니티 적용: 3D 공간의 X, Y, Z 좌표를 나타내는 핵심 단위인 Vector3가 바로 구조체로 이루어져 있습니다.

튜플 (Tuple): 여러 개의 필드를 임시로 묶어 반환하거나 다룰 때 사용하는 가벼운 데이터 구조입니다.



using System;

namespace MakeAssassin
{
    class Character
    {
        public string Name { get; set; }
        public int Hp { get; set; }

        // 생성자
        public Character(string Name, int Hp)
        {
            this.Name = Name;
            this.Hp = Hp;
        }

        // 메소드
        public void TakeDamage(int damage)
        {
            Hp -= damage;
            Console.WriteLine($"{Name} 이(가) {damage}의 데미지를 입었습니다. (남은체력: {Hp})");
        }
    }

    class Assassin : Character
    {
        public Assassin(string Name, int Hp) :base(Name, Hp) // 이름과 체력을 입력받아 부모 클래스의 생성자로 넘겨줌
        {          
        }

        public void Assassinate()
        {
            Console.WriteLine($"{Name} 이(가) 은신후 치명적인 암살 공격을 시도합니다!");
        }
    }
    

    class MainApp
    {
        public static void Main()
        {
            Assassin assassin = new Assassin("에지오", 100);

            assassin.TakeDamage(30);

            assassin.Assassinate();
        }
    }

}
